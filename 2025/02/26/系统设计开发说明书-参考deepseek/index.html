个性化景区推荐系统设计说明书

1. 系统概述

1.1 项目背景

  • 需求痛点：传统景区推荐系统存在冷启动严重、无法动态适应用户兴趣迁移等问题。
  • 目标用户：旅游爱好者、景区管理人员、本地游客。
  • 核心价值：通过个性化推荐提升用户体验，帮助景区优化资源配置。

1.2 系统功能

功能模块 描述
实时推荐 基于用户位置和行为的动态推荐
旅行路线规划 景点关联关系挖掘（知识图谱增强）
用户画像管理 基于RFM模型的用户兴趣分析
冷启动解决方案 结合POI类别和知识图谱的扩展推荐

层级 技术组件 说明
前端 Vue.js + Leaflet.js + ECharts 用户界面、地图可视化、数据展示
后端 Django + Django REST Framework 业务逻辑处理、API接口开发
算法层 Node2Vec + KGAT + FTRL 推荐算法核心模块
数据层 PostgreSQL + PostGIS + Neo4j 数据存储与空间查询
基础设施 Redis + Docker + Nginx 缓存、容器化部署、负载均衡
